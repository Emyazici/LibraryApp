║  HATA 3 — Loan State Machine Kırık                        [ İŞ KURALI ]    ║
║  Dosya  : Domain/Entities/Loan.cs  (satır 54-63)                           ║
╚══════════════════════════════════════════════════════════════════════════════╝

  MEVCUT (YANLIŞ):
  ─────────────────
    public void Return()
    {
        if (Period.IsOverdue())
        {
            Status = LoanStatus.Overdue;             // Overdue'ya alındı
            Fee = Money.Create(CalculateFee(), "TRY");
            // ← Burada bitiyor. Gecikmiş kitap sonsuza kadar Overdue kalıyor!
        }
        else
        {
            Status = LoanStatus.Returned;
        }
    }

  OLMASI GEREKEN:
  ────────────────
    public void Return()
    {
        if (Status == LoanStatus.Returned)
            throw new BusinessRuleException("Bu ödünç zaten iade edilmiş.");

        ActualReturnDate = DateTime.UtcNow;

        if (Period.IsOverdue())
            Fee = Money.Create(CalculateFee(), "TRY");

        Status = LoanStatus.Returned;                // Her iki durumda Returned
        AddDomainEvent(new LoanReturnedEvent(Id, BookId, MemberId));
    }

║  HATA 5 — CalculateFee() Side Effect                        [ TASARIM ]    ║
║  Dosya  : Domain/Entities/Loan.cs  (satır 41-45)                           ║
╚══════════════════════════════════════════════════════════════════════════════╝

  SORUN: Hesaplama metodu hem değer döndürüyor hem de OverDueDays state'ini
  değiştiriyor. Bir query metodu state mutasyonu yapmamalı.

  MEVCUT (YANLIŞ):
    public int CalculateFee()
    {
        var overdueDays = (int)(DateTime.UtcNow - Period.ExpectedReturnDate).TotalDays;
        OverDueDays = overdueDays;    // ← Side effect!
        return overdueDays > 0 ? overdueDays * 2 : 0;
    }

  OLMASI GEREKEN:
    public int CalculateFee()
    {
        var overdueDays = (int)(DateTime.UtcNow - Period.ExpectedReturnDate).TotalDays;
        return overdueDays > 0 ? overdueDays * 2 : 0;
        // OverDueDays'i Return() içinde set et; hesaplama saf kalır.
    }
